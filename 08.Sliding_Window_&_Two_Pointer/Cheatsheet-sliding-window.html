1
Fixed window size
window size = k
Pattern
Window size is always k. No shrink logic needed — l is always r − k + 1. Slide the window and maintain your aggregate (sum, freq map, deque).
Template
def fixed_window(nums, k):
    window_sum = sum(nums[:k])
    ans = window_sum

    for r in range(k, len(nums)):
        window_sum += nums[r]
        window_sum -= nums[r - k]  # drop leftmost
        ans = max(ans, window_sum)  # or min / record
    return ans
Recognition: "subarray / substring of size k", "every k elements", "k consecutive"
Variant — sliding window max (deque)
from collections import deque

dq = deque()  # stores indices, front = max

for r in range(len(nums)):
    # pop smaller elements from back
    while dq and nums[dq[-1]] <= nums[r]:
        dq.pop()
    dq.append(r)

    # remove out-of-window index
    if dq[0] < r - k + 1:
        dq.popleft()

    if r >= k - 1:
        ans.append(nums[dq[0]])

2
Variable window — longest / maximize
shrink when invalid
Pattern
Expand right freely. Only shrink from the left when the window breaks the constraint. After shrinking, the window is valid again — record the longest valid window seen. 
def longest_window(s, k):
    freq = {}
    l = 0
    ans = 0

    for r in range(len(s)):
        freq[s[r]] = freq.get(s[r], 0) + 1     # expand

        while <window is invalid>:             # too many distinct / sum > k etc
            freq[s[l]] -= 1
            if freq[s[l]] == 0: del freq[s[l]]
            l += 1                               # shrink

        ans = max(ans, r - l + 1)               # window is now valid, record
    return ans
Constraint: no repeats
Invalid when any freq > 1. LC 3.
Constraint: at most k distinct
Invalid when len(freq) > k. LC 340, LC 159.
Constraint: k replacements
Invalid when (r−l+1) − maxFreq > k. LC 424.
Recognition: "longest", "maximum length", "at most k [distinct / zeros / replacements]"

3
Variable window — minimum / shortest
shrink when valid
Pattern — the flip
The shrink condition is reversed compared to pattern 2. Expand until the window satisfies the condition, then shrink from the left to minimize it, recording each valid window before removing the left element. 
def minimum_window(s, t):
    need = {}
    for c in t: need[c] = need.get(c, 0) + 1
    have, required = 0, len(need)
    window = {}; l = 0; ans = ""

    for r in range(len(s)):
        window[s[r]] = window.get(s[r], 0) + 1    # expand
        if s[r] in need and window[s[r]] == need[s[r]]:
            have += 1

        while have == required:                   # valid! shrink to minimize
            if not ans or r-l+1 < len(ans):
                ans = s[l:r+1]                    # record BEFORE shrinking
            if s[l] in need:
                if window[s[l]] == need[s[l]]: have -= 1
            window[s[l]] -= 1
            l += 1
    return ans
Key difference from longest: you update the answer INSIDE the shrink loop, not after it. You shrink while valid, recording each improvement.

Count subarrays — at most k
count += r − l + 1
The key insight
After shrinking so the window [l..r] is valid, every subarray ending at r and starting anywhere from l to r is also valid (fewer elements = easier to satisfy). That's r−l+1 subarrays counted in one line. This makes it O(n) instead of O(n²). 
def at_most(arr, k):
    from collections import defaultdict
    freq = defaultdict(int)
    distinct = 0
    count = 0
    l = 0

    for r in range(len(arr)):
        freq[arr[r]] += 1
        if freq[arr[r]] == 1:         # new element entered window
            distinct += 1

        while distinct > k:            # shrink until valid
            freq[arr[l]] -= 1
            if freq[arr[l]] == 0:
                distinct -= 1
            l += 1

        count += r - l + 1            # ALL subarrays ending at r

    return count
The shrink condition is > k (not >= k). Using >= would shrink too far and give you atMost(k−1) by accident.

Count subarrays — exactly k
atMost(k) − atMost(k−1)
Why you can't count exactly k directly
"Exactly k" is not monotone — expanding can break the condition, shrinking can also break it. There's no clean invariant. Instead: atMost(k) counts windows with 1,2,…k. atMost(k−1) counts 1,2,…k−1. Their difference is exactly k. 
def exactly_k(arr, k):
    return at_most(arr, k) - at_most(arr, k - 1)

# Adapt at_most for different constraints:
#   distinct count → track len(freq)
#   odd numbers   → track count of odds
#   zeros         → track count of zeros
# Nice subarrays (k odds) variant:
def number_of_subarrays(nums, k):
    def at_most_odds(k):
        odds = 0; count = 0; l = 0
        for r in range(len(nums)):
            if nums[r] % 2 == 1:
                odds += 1
            while odds > k:
                if nums[l] % 2 == 1: odds -= 1
                l += 1
            count += r - l + 1
        return count
    return at_most_odds(k) - at_most_odds(k-1)

Count subarrays — at least k & prefix sum
complement or hashmap
At least k — two approaches
Mirror of atMost
Once the window satisfies "at least k", extending right always keeps it valid. So count += n − r for every valid starting position. 
# Option A: direct count
while valid:
    count += n - r    # all right extensions
    # remove arr[l]
    l += 1

# Option B: complement (simpler)
def at_least(arr, k):
    n = len(arr)
    total = n * (n + 1) // 2
    return total - at_most(arr, k - 1)
Prefix sum — when window breaks
Use when you have negatives
With negative numbers, shrinking a window can increase the sum — the window is no longer monotone. Sliding window fails. Use prefix sums instead. 
def subarray_sum_k(nums, k):
    prefix_count = {0: 1}   # init: empty prefix
    prefix = 0; count = 0

    for num in nums:
        prefix += num
        count += prefix_count.get(prefix-k, 0)
        prefix_count[prefix] = \
            prefix_count.get(prefix, 0) + 1

    return count

# sum(l..r) = prefix[r] - prefix[l-1] = k
# → prefix[l-1] = prefix[r] - k
# → count past prefixes = current - k
The prefix_count[0] = 1 initialization is not optional. Without it, subarrays that start from index 0 (where prefix[l-1] = 0) are missed entirely.
